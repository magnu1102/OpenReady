# Contributing to OpenReady

Thanks for your interest in OpenReady! This project analyzes GitHub repositories with a
deterministic, local-first engine and a calm desktop UI. Contributions of all sizes are welcome —
bug reports, docs, tests, and features.

Please also read the [product principles](docs/product-principles.md); they explain the values
(deterministic, transparent, local-first, no required AI/accounts) that any change should respect.

## Prerequisites

- [Node.js](https://nodejs.org/) 20+
- [pnpm](https://pnpm.io/) 9+
- [Rust](https://rustup.rs/) (stable) — only needed for the desktop shell (`pnpm tauri dev`)

## Getting started

```bash
pnpm install
pnpm dev          # frontend only, in your browser (fastest iteration)
pnpm tauri dev    # full desktop app (requires Rust)
```

The CLI runs the same analyzer outside the shell:

```bash
pnpm cli -- analyze octocat --limit 5
```

## Quality gates

Every change must pass the same checks CI runs (`.github/workflows/lint-and-test.yml`):

```bash
pnpm lint           # ESLint
pnpm typecheck      # tsc --noEmit (app)
pnpm typecheck:cli  # tsc --noEmit (CLI tsconfig)
pnpm test           # Vitest
pnpm build          # production web build
pnpm build:cli      # esbuild CLI bundle
```

`pnpm format` applies Prettier. A pre-commit hook runs ESLint/Prettier on staged files.

The CLI bundle `dist-cli/openready.mjs` is committed (the GitHub Action runs it with zero
install). When a change touches CLI sources or bumps esbuild, run `pnpm build:cli` and commit
the rebuilt bundle — CI fails on a stale artifact.

## Pull requests

1. Branch off `main` (e.g. `fix/cli-arg-parsing`).
2. Keep commits focused; write clear, imperative commit subjects (the history loosely follows
   [Conventional Commits](https://www.conventionalcommits.org/): `fix:`, `feat:`, `docs:`, …).
3. Add or update tests for behavior changes, and a `CHANGELOG.md` entry under `[Unreleased]`.
4. User-facing strings live in `src/lib/copy.ts` and follow the
   [voice and tone guide](docs/voice-and-tone.md); visual changes follow the
   [Aurora design system](docs/design-system.md).
5. Make sure all quality gates above pass locally.
6. Open a PR against `main` and fill in the template.

## Reporting bugs and requesting features

Use the issue templates under **Issues → New issue**. For security issues, do **not** open a public
issue — see [SECURITY.md](SECURITY.md).

## Authorship and AI-tool policy

OpenReady is the maintainer's own public portfolio and open-source project. To keep authorship
honest:

- **Do not** add AI tools or models (Claude, ChatGPT, Codex, Copilot, OpenAI, Anthropic, etc.) as
  contributors, co-authors, maintainers, or collaborators.
- **Do not** add AI-attribution lines to commits, pull requests, code comments, or documentation
  (no `Co-authored-by:` for AI tools, no "Generated by …" footers, no AI metadata).

You are welcome to _use_ AI tools to help write your contribution — just submit the work under your
own name without AI attribution.
