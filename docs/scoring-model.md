# Scoring model

Stub — full design lands in Phase 5.

Planned principles:

- Scores are evidence-based. Each check resolves to one of `passed`, `failed`, `not-applicable`, `unknown`.
- Scores roll up by category (Documentation, Presentation, Buildability, Maintainability, Testing/CI, Deployment/Operations, Metadata/Discoverability, Security hygiene).
- Every total score must be inspectable down to the individual check that contributed.
- Human-readable labels (Portfolio-ready, Almost ready, Needs documentation, …) accompany numeric scores.
- Custom scoring rules and project-type-aware adjustments are later additions.
